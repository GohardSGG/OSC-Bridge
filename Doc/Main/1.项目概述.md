# 1. 项目概述 (Project Overview)

## 1.1 项目简介
**OSC-Bridge** 是一个跨平台的（当前专注于 Windows）高性能 OSC (Open Sound Control) 桥接工具。它的核心目标是打通不同音乐/视觉软件（如 Ableton Live, TouchDesigner, VRChat）与 Web 前端或其他网络设备之间的通信壁垒。

通过结合 Rust 的高性能底层和 Svelte 的现代化响应式 UI，OSC-Bridge 提供了极低延迟的消息转发、实时流量监控以及灵活的配置管理。

## 1.2 核心价值
- **高性能桥接**：利用 Rust 异步运行时 (Tokio) 处理高并发 OSC 数据包。
- **双向通信**：支持 OSC -> WebSocket 和 WebSocket -> OSC 的双向数据流。
- **即时反馈**：内置流量监视器 (Traffic Monitor) 实时可视化数据吞吐量。
- **非侵入式后台**：支持最小化到系统托盘，静默运行，且具备开机自启能力（基于 Windows Task Scheduler）。

## 1.3 目标用户
- **新媒体艺术家**：需要将传感器数据或音频信号桥接到 Web 界面进行展示。
- **虚拟偶像 (Vtuber) 技术人员**：实现直播间互动指令控制 OSC 设备。
- **主要开发者**：接手本项目的后续维护人员。

## 1.4 核心功能清单
| 功能模块 | 描述 |
| :--- | :--- |
| **OSC 监听** | 支持多端口监听 (如 9000, 9001)，自动聚合数据。 |
| **OSC 转发** | 将接收到的 OSC 消息转发到指定 IP:Port。 |
| **WebSocket 主机** | 内置 WebSocket 服务器，允许前端页面订阅实时数据。 |
| **系统设置** | 端口管理、转发目标管理、界面缩放、语言切换。 |
| **流量监控** | 实时波形图展示每秒接收和发送的数据包数量。 |
